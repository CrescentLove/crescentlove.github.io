shell脚本

经常在开发机和SPR上来回传代码和模块，所以写个脚本方便摸鱼

脚本基本就几步

  1. #!/bin/bash在开头声明使用的解释器，约定俗成

  2. 定义相关变量

  3. 定义函数

  4. 跑main逻辑

变量

传参

有点像汇编啊格式化啊c++包装器function啊里面占位符一样

通过$1/$2/$n...来调用脚本传进来的外部参数，$0表示脚本名（包含路径），其他用法。

函数

没有return时返回最后一条命令的运行结果，通过$?可以查看最近一条命令的运行结果，0表示没有错误

一定要在调用前定义函数，所以总是放在全局变量的后面定义，避免出错

函数传参形式同脚本传参

if .. fi

for/while

case .. esac

附

特殊字符

& | < > * ? ' " ; [ - ] # \

正则的小部分知识（查表完事记两个常用即可）

*用来匹配任意字符串（可为空）

?用来匹配单个字符（不可为空），且? *均不能匹配/

[ - ]用来匹配方括号里的字符，hyphen用来表示字符范围（仅在方括号内生效）

目录结构

.. ./ / ~
向上一级根目录 当前绝对路径 根目录 当前用户主目录

尽量使用以./xxx或者/xxx开头的绝对路径，避免相对路径优先在预设的搜索路径里查找文件

文件权限

按照文件主，组用户和其他用户的分类

user group others
rwx rwx rwx

rwx按二进制赋权再按8|10进制转化为权限码

比如rw-r--r--按二进制赋权421再转化为8进制即为644

比如rwxrwxrwx按二进制赋权421再转化为8进制即为777

文件描述符

0 1 2
std in std out std err

输入输出重定向

> < >> >& <&
输出重定向 输入重定向 追加输出 输出合并 输入合并

Eg:commandexpl > /dev/null 2>&1能够将stdout&stderr全部屏蔽输出

参数处理

$? $# $$/$!
最近命令退出状态 传参总数 当前脚本/后台最新的进程ID
