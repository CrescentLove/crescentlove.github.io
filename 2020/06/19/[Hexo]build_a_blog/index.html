基于hexo的静态博客搭建

发表于 2020-06-19 分类于 个人向手册
记录手册，详细还是看官方文档吧

一、安装依赖库

  • node.js

  • git

git初始配置及常用命令行可以查阅菜鸟教程的手册：git

另外需要注意的是，npm尽量添加镜像源加速，不然后面下载hexo会非常非常令人捉急。

二、安装hexo

  • 创建本地文件夹作为博客源，比如C:\Users\RAM\crescentBlog，日后hexo配置文件以及个人文章都将放入其中

  • 用cmd一路cd到你创建的文件夹之中，之后所有命令均需要在此路径下

  • 在该文件路径下安装hexo，使用命令行npm install hexo-cli -g

    PS：一定要在该文件夹下安装hexo，网上很多教程都先安装后建文件夹，在init初始化的时候就会报错提示node_modules找不到

  • 初始化hexo博客，使用命令行hexo init

  • 可以使用hexo s并访问http://localhost:4000来初步浏览生成的静态博客

三、推送到gitee

  • 在gitee创建一个仓库名和用户地址一致的仓库，比如https://gitee.com/crescentlove/crescentlove
  • 具体gitee食用请查阅gitee指南
  • 打开文件夹根目录下的_config.yml文件，修改deploy配置以及网站地址（即https://仓库名.gitee.io）

  • 文件夹根目录打开cmd，安装部署插件并推送博客到gitee。

  • 接下来打开gitee仓库，在服务项打开GiteePages服务。

  • 之后基本就只需要以下三条操作，然后在page页手动更新。
